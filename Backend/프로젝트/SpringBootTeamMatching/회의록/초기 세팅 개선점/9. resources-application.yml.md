## ✅ 줄별 역할/기능 해설

```yaml
spring:
  profiles:
    active: local  # 현재 활성 프로파일을 local로 설정.
                   # application-local.yml이 존재하면 해당 설정이 함께 적용됨.
                   # 배포 시 prod 등으로 변경 가능.

  datasource:
    url: ${DB_URL}
    # DB 접속 URL. 환경변수 DB_URL에서 값을 읽어옴.
    # 예: jdbc:postgresql://localhost:5432/moyeo

    username: ${DB_USERNAME}
    # DB 접속 계정명. 환경변수에서 주입.

    password: ${DB_PASSWORD}
    # DB 접속 비밀번호. 코드에 하드코딩하지 않고 환경변수로 관리.

    driver-class-name: org.postgresql.Driver
    # PostgreSQL JDBC 드라이버 지정.

  jpa:
    hibernate:
      ddl-auto: none
      # Hibernate가 테이블을 자동 생성/수정하지 않도록 설정.
      # none → 스키마 관리는 Flyway가 담당.
      # (create, update, validate 등 옵션 있음)

    properties:
      hibernate:
        format_sql: true
        # Hibernate가 출력하는 SQL을 보기 좋게 포맷팅.
        # 콘솔 로그 가독성 향상용.

  flyway:
    enabled: true
    # Flyway 마이그레이션 활성화.
    # 애플리케이션 시작 시 db/migration 경로의 SQL 실행.

    baseline-on-migrate: true
    # 기존 DB가 있을 때 baseline을 자동 생성.
    # 초기 도입 시 충돌 방지용.

    locations: classpath:db/migration
    # Flyway가 마이그레이션 파일을 찾는 경로 지정.
    # src/main/resources/db/migration

  security:
    oauth2:
      client:
        registration:
          google:
            client-id: ${GOOGLE_CLIENT_ID}
            # Google OAuth2 클라이언트 ID (환경변수 주입)

            client-secret: ${GOOGLE_CLIENT_SECRET}
            # Google OAuth2 클라이언트 Secret (환경변수 주입)

            scope:
              - openid
              # OpenID Connect 사용 (idToken 발급 필수)

              - email
              # 사용자 이메일 정보 요청

              - profile
              # 기본 프로필 정보 요청 (name, picture 등)

app:
  oauth2:
    redirect-url: ${OAUTH2_REDIRECT_URL}
    # OAuth2 로그인 성공 후 리다이렉트할 프론트엔드 URL.
    # 예: http://localhost:3000/oauth/callback

  jwt:
    secret: ${JWT_SECRET}
    # JWT 서명에 사용할 비밀 키.
    # 반드시 충분히 길고 랜덤한 문자열이어야 함.

    access-exp-min: 60
    # Access Token 만료 시간 (분 단위).
    # 현재 60분으로 설정.

server:
  port: ${SERVER_PORT:8080}
  # 서버 포트 설정.
  # SERVER_PORT 환경변수가 없으면 기본 8080 사용.
```

# 🔴 P0 — 반드시 점검/개선 (실무 기준 치명적 리스크 가능)

---

## 1️⃣ JWT Secret 길이/강도 확인 필수

```yaml
app:
  jwt:
    secret: ${JWT_SECRET}
```

⚠️ 문제 가능성:

- 짧거나 단순 문자열이면 JWT 위변조 가능성 증가.
    
- 최소 256bit 이상 랜덤 키 권장 (HS256 기준).
    

실무 권장:

- 32바이트 이상 랜덤 문자열
    
- Base64 인코딩 키 사용
    

---

## 2️⃣ OAuth2 로그인 방식 혼합 가능성

현재:

```yaml
spring.security.oauth2.client.registration.google
```

→ Spring Security OAuth2 리다이렉트 플로우용 설정.

그런데 만약:

- 프론트에서 idToken 받아 POST 방식 사용한다면
    
- client-secret은 필요 없음
    
- oauth2Login 설정 자체가 불필요
    

⚠️ 구조 혼합 여부 반드시 확인 필요.

---

## 3️⃣ baseline-on-migrate 주의

```yaml
baseline-on-migrate: true
```

- 기존 DB에 마이그레이션 이력 없을 때 baseline 자동 생성.
    
- 운영 DB에 실수로 적용 시 마이그레이션 누락 가능.
    

졸업 프로젝트면 OK  
실서비스면 신중 사용.

---

# 🟡 P1 — 실무 권장 (졸업 프로젝트에서는 선택)

---

## 4️⃣ access-exp-min 60분은 다소 긴 편

```yaml
access-exp-min: 60
```

실무 권장:

- 15~30분
    
- Refresh Token 별도 운영
    

졸업 프로젝트면 60분도 무방.

---

## 5️⃣ 로그 설정 미흡

SQL 로그 활성화 옵션 추가 가능:

```yaml
logging:
  level:
    org.hibernate.SQL: debug
```

개발 환경에서 유용.

---

## 6️⃣ Timezone 설정 없음

실무에서는:

```yaml
spring:
  jackson:
    time-zone: UTC
```

또는 서버 timezone 고정.

졸업 프로젝트면 필수 아님.


조정 필요.

# 📌 핵심 정리

이 설정 파일은:

✔ 환경변수 기반으로 잘 구성되어 있음  
✔ Flyway + ddl-auto none 구조 적절  
✔ JWT secret 강도만 반드시 점검 필요  
✔ OAuth2 구조 혼합 여부 확인 필요
