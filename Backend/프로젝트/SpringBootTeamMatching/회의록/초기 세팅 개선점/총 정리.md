# 1) OAuth2 세션 정책 수정 (P0)

### 📁 경로

`src/main/java/com/catholic/moyeo/config/WebSecurityConfig.java`

### ❗ 이유

`oauth2Login`은 로그인 시작 시 만든 `state` 등을 저장했다가 콜백에서 검증해야 해서, 기본 구현은 **세션을 잠깐 사용**하는 경우가 많음.  
`STATELESS`로 고정하면 로그인 플로우가 깨질 수 있음.

### ✅ 수정안 (변경 구간만)

**기존**

```java
.sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
```

**변경**

```java
.sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED))
```

# 2) CORS 과개방 수정 (P0)

### 📁 경로

`src/main/java/com/catholic/moyeo/config/WebSecurityConfig.java`  
`src/main/resources/application-local.yml`  
`src/main/resources/application-prod.yml`

### ❗ 이유

`addAllowedOriginPattern("*")` + `allowCredentials(true)`는 사실상 “모든 출처에 자격증명 허용”이라 위험.  
너희는 **환경별로 프론트 도메인이 명확**하니까 화이트리스트로 제한해야 함.

### ✅ 수정안 (변경 구간만)

#### 2-1) WebSecurityConfig.java에 allowed origins 주입

**추가**

```java
@Value("${app.cors.allowed-origins}")
private List<String> allowedOrigins;
```

#### 2-2) corsConfigurationSource() 변경

**기존**

```java
config.addAllowedOriginPattern("*");
config.addAllowedMethod("*");
config.addAllowedHeader("*");
config.setAllowCredentials(true);
```

**변경**

```java
config.setAllowedOrigins(allowedOrigins);
config.setAllowedMethods(List.of("GET","POST","PUT","PATCH","DELETE","OPTIONS"));
config.setAllowedHeaders(List.of("Authorization","Content-Type"));
config.setAllowCredentials(true);
```

#### 2-3) application-local.yml 추가

`src/main/resources/application-local.yml`

**추가**

```yaml
app:
  cors:
    allowed-origins:
      - http://localhost:3000
```

#### 2-4) application-prod.yml 추가

`src/main/resources/application-prod.yml`

**추가**

```yaml
app:
  cors:
    allowed-origins:
      - https://moyeo.com
```

# 3) OAuth 관련 permitAll 정합성 (P0)

### 📁 경로

`src/main/java/com/catholic/moyeo/config/WebSecurityConfig.java`

### ❗ 이유

서버 주도 OAuth2면 최소한 아래 경로는 인증 없이 열려야 함:

- 로그인 시작: `/oauth2/**`
    
- 콜백 처리: `/login/oauth2/**`  
    그리고 너희가 환경변수로 쓰는 테스트 콜백이 `/oauth/callback`이면 이건 permitAll로 열어야 로컬 단독 테스트가 편함.
    

### ✅ 수정안 (변경 구간만)

**기존 permitAll 목록에 추가/정리**

```java
.requestMatchers(
    "/",
    "/oauth2/**",
    "/login/oauth2/**",
    "/oauth/callback"
).permitAll()
```

> 참고: 지금 목록에 `/oauth/**`, `/login/**` 같은 게 섞여있는데, 실제 사용하는 경로 기준으로 줄이는 게 좋음.

# 4) JWT 필터 관련 코드 “지금은 주석/보류” (P0)

### 📁 경로

`src/main/java/com/catholic/moyeo/config/WebSecurityConfig.java`

### ❗ 이유

`JwtAuthenticationFilter`를 아직 안 만들었으면, 의존성 주입/필터 등록이 있으면 **컴파일 자체가 깨짐**.  
그래서 JWT를 만들기 전 단계에서는 해당 부분은 보류해야 함.

### ✅ 수정안 (변경 구간만)

- 아래 두 부분이 이미 있다면 **JWT 만들기 전까지 주석**:
    

1. 필드 주입
    

```java
private final JwtAuthenticationFilter jwtAuthenticationFilter;
```

2. 필터 등록
    

```java
.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
```

# 5) DB 마이그레이션 정합성 (P0)

### 📁 경로

`src/main/resources/db/migration/V2_create_app_user.sql`

### ❗ 이유

엔티티에서 `created_at`, `updated_at`은 `nullable = false`인데 DB가 NOT NULL이 아니면 정합성이 깨짐(특히 운영에서).

### ✅ 수정안 (변경 구간만)

**기존**

```sql
created_at TIMESTAMP DEFAULT now(),
updated_at TIMESTAMP DEFAULT now()
```

**변경**

```sql
created_at TIMESTAMP NOT NULL DEFAULT now(),
updated_at TIMESTAMP NOT NULL DEFAULT now()
```

# 7) /api/me 반환 타입 안정화 (P1)

### 📁 경로

`src/main/java/com/catholic/moyeo/MeController.java`

### ❗ 이유

현재 `Object` 반환은 principal 타입이 바뀌면 API 계약이 흔들림. DTO로 고정이 낫다.

### ✅ 수정안 (변경 구간만)

- DTO 추가 + 반환 타입 변경
    

```java
public record MeResponse(Long memberId) {}

@GetMapping("/api/me")
public MeResponse me(Authentication authentication) {
    Long memberId = (Long) authentication.getPrincipal();
    return new MeResponse(memberId);
}
```
