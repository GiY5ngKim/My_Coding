## ✅ 줄별 역할/기능 해설

```java
package com.catholic.moyeo; 
// 애플리케이션의 루트 패키지 선언.
// @SpringBootApplication이 붙은 클래스가 이 패키지 아래를 기본 Component Scan 대상으로 삼는다.

import org.springframework.boot.SpringApplication;
// Spring Boot 애플리케이션을 실행(run)하기 위한 진입점 유틸 클래스.

import org.springframework.boot.autoconfigure.SpringBootApplication;
// Spring Boot 메인 어노테이션.
// 내부적으로 @Configuration(설정 클래스), @EnableAutoConfiguration(자동 설정 활성화),
// @ComponentScan(컴포넌트 스캔) 기능을 포함한다.

import org.springframework.data.jpa.repository.config.EnableJpaAuditing;
// Spring Data JPA Auditing 기능을 활성화하기 위한 어노테이션 임포트.
// (@CreatedDate, @LastModifiedDate 등이 자동으로 채워지게 하는 기능)

@EnableJpaAuditing
// JPA Auditing 기능 활성화.
// 엔티티에 @CreatedDate / @LastModifiedDate가 붙은 필드가 있을 때
// 엔티티 저장/수정 시점에 자동으로 값을 세팅해준다.
// (전제: 엔티티에 @EntityListeners(AuditingEntityListener.class) 적용되어 있거나,
// Spring Data JPA Auditing이 감지할 수 있는 설정이 되어 있어야 함)

@SpringBootApplication
// Spring Boot 애플리케이션의 시작점임을 선언.
// 자동 설정 + 컴포넌트 스캔 + 설정 클래스 역할을 수행한다.
public class MoYeoApplication {

	public static void main(String[] args) {
        // Java 프로그램의 진입점(main 메서드).
        // JVM이 애플리케이션 실행 시 가장 먼저 호출한다.

		SpringApplication.run(MoYeoApplication.class, args);
        // Spring Boot 애플리케이션을 실행한다.
        // - 스프링 컨테이너(ApplicationContext) 생성
        // - 컴포넌트 스캔으로 @Component/@Service/@Repository/@Controller/@Configuration 등 Bean 등록
        // - AutoConfiguration 적용
        // - 내장 웹서버(Tomcat 등) 기동
        // - 애플리케이션 이벤트 라이프사이클 수행
	}

}
```

# 🔴 P0 — 반드시 점검

---

## 1️⃣ @EnableJpaAuditing 정상 위치

현재 위치:

```java
@EnableJpaAuditing
@SpringBootApplication
public class MoYeoApplication
```

✔ 루트 패키지에 위치하므로 정상 동작.

❗ 단, 아래 조건 확인 필요:

- `createdAt`, `updatedAt` 필드가 `null`로 저장되지 않는지 테스트 필요.
    
- DB 컬럼 nullable=false라면 auditing 미작동 시 바로 insert 실패.
    

---

## 2️⃣ AuditorAware 미구현 상태

현재는 단순 CreatedDate / LastModifiedDate만 사용.

만약 향후 다음이 필요하면:

```
createdBy
updatedBy
```

→ `AuditorAware` 구현 필요.

지금은 없어도 문제 없음.

# 🟡 P1 — 실무 권장

---

## 3️⃣ 시간대(TimeZone) 설정 명확화

현재 LocalDateTime 사용.

실무에서는 보통:

- 서버 timezone 고정 (UTC)
    
- DB timezone 고정
    
- 또는 OffsetDateTime 사용
    

졸업 프로젝트에서는 LocalDateTime 충분.

---

## 4️⃣ 애플리케이션 클래스에 설정을 몰아넣지 않기

현재는 간단해서 문제 없음.

실무에서는:

- AuditingConfig
    
- SecurityConfig
    
- JpaConfig
    

처럼 분리하는 경우 많음.

지금은 유지해도 무방.
