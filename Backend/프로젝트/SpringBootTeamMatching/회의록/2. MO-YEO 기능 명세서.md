## 1. 서비스 개요

- 서비스 형태: **웹앱(반응형)**
- 대상 사용자: **가톨릭대학교 구성원** -> 각 학교 이메일 주소로 분기처리
- 접근 방식: 홈 화면에서 주요 기능 버튼을 통해 각 기능 페이지로 이동

---

## 2. 홈(Home) 화면 구성

1. 팀원 찾기
2. 프로젝트 모집
3. 공모전 모집
4. 스터디 모집
5. 자유게시판(고민/질문)

※ 버튼 클릭 시 각 기능의 목록 페이지로 이동

---

## 3. 로그인 / 회원가입 정책

### 3-1. 로그인 방식

- **OAuth 로그인 방식 사용**

### 3-2. 이메일 도메인 제한

- OAuth 로그인 후 이메일 주소 검증 수행
- 이메일 도메인이 **`@catholic.ac.kr`이 아닌 경우 로그인 불가**
- 검증은 서버에서 최종 처리

---

## 4. 팀원 찾기 기능

### 4-1. 팀원 목록

- 팀원 카드 리스트 형태로 제공
- 카드 정보:
    - 이름/닉네임
    - 학과
    - 역할(part)
    - 학년
    - 자기소개 요약
    - 기술 스택(컴공 이외 학과 필터링 불가?)

### 4-2. 필터

- 필터:
    - 역할(part)
    - 학과
    - 학년
    - 기술 스택

### 4-3. 프로필 상세

- 팀원 카드 클릭 시 프로필 상세 페이지로 이동

---

## 5. 모집 기능 (카테고리별 분리)

### 5-1, 5-2. 프로젝트, 공모전 모집

- 모집글 CRUD 기능 제공(작성, 조회, 수정, 삭제)
- 입력 항목:
    - 제목
    - 설명
    - 모집 역할 및 인원
    - 기술 스택
    - 모집 상태(모집중/마감)
- 모집글에 **연락수단 필수 포함 → (1:1 채팅 구현 가능?)**

---

### 5-3. 스터디 모집

- 모집글 CRUD 기능 제공(작성, 조회, 수정, 삭제)
- 입력 항목:
    - 제목
    - 내용
    - 모집 인원
    - 기간
    - 모집 상태(모집중/마감)
- 모집글에 **연락수단 필수 포함 → (1:1 채팅 구현 가능?)**

---

## 6. 자유게시판 기능

- 전 학과 공용 게시판
- 입력 항목:
    - 제목
    - 설명
    - 모집 역할 및 인원
    - 기술 스택
    - 모집 상태(모집중/마감)

### 6-1. 게시글(전체 열람 가능 학과는 필터링만)

- 게시글 CURD 기능 제공
- 게시글 작성 시 학과 온보딩 적용(모든 작성자는 본인 학과로 자동 분류)
- 입력 항목:
    - 제목
    - 본문
- 좋아요 기능

### 6-2. 댓글

- 댓글 CURD 기능 제공
- 입력 항목:
    - 본문
- 좋아요 기능

### 6-3. 검색

- 제목 및 본문 키워드 검색
- 학과 필터링

## 7. 추가 확장 기능

- 1:1 채팅

## 8. 확정 사항 요약

- 홈 화면은 기능 버튼 중심 구조로 구성
- 로그인은 OAuth 사용 + 학교 이메일 도메인 제한
- 모집 기능은 프로젝트 / 공모전 / 스터디로 구분
- 스터디 및 자유게시판은 단순 게시판 구조 유지
- 자유게시판은 컴공 외 학과도 사용 가능하도록 설계
