## 1. 서비스 개요

- 서비스 형태: **웹앱 (반응형)**
- 대상 사용자: **가톨릭대학교 구성원**
- 로그인 조건: **OAuth 로그인 + 학교 이메일 도메인 제한**
- 접근 방식: 홈 화면에서 주요 기능 버튼을 통해 각 기능 페이지로 이동

---

## 2. 홈(Home) 화면 구성

홈 화면은 **기능 진입 중심 구조**로 구성한다.

### 주요 기능 버튼

1. 팀원 찾기
2. 프로젝트 모집
3. 공모전 모집
4. 스터디 모집
5. 자유게시판

> 각 버튼 클릭 시 해당 기능의 **목록 페이지**로 이동한다.

---

## 3. 로그인 / 회원 정책

### 3-1. 로그인 방식

- OAuth 로그인 방식 사용
- 서버에서 로그인 성공 시 **JWT 발급**

### 3-2. 이메일 도메인 제한

- OAuth 로그인 후 이메일 주소 검증
- 이메일 도메인이 `@catholic.ac.kr`이 아닌 경우 **로그인 불가**
- 이메일 검증은 **서버에서 최종 처리**

---

## 4. 팀원 찾기 기능

### 4-1. 팀원 목록 조회

- 팀원 카드 리스트 형태로 제공
- 카드 정보:
    - 닉네임
    - 학과
    - 역할(part)
    - 자기소개 요약
    - 기술 스택

### 4-2. 필터 기능

- 역할(part)
- 학과
- 기술 스택

### 4-3. 프로필 상세 조회

- 팀원 카드 클릭 시 프로필 상세 페이지로 이동
- 상세 정보:
    - 자기소개
    - 기술 스택
    - GitHub 링크 (선택)

---

## 5. 모집 기능 (카테고리별 분리)

모집 기능은 **프로젝트 / 공모전 / 스터디**로 구분되며,

도메인 구조와 API는 공통 모집 모델을 기반으로 한다.

---

### 5-1. 프로젝트 / 공모전 모집

### 제공 기능

- 모집글 CRUD (작성 / 조회 / 수정 / 삭제)
- 모집 상태 변경 (모집중 / 마감)
- 모집 지원 / 지원 취소
- 지원자 목록 조회 (작성자 전용)

### 모집글 입력 항목

- 제목
- 설명
- 모집 역할 및 인원
- 기술 스택
- 모집 상태
- 연락 수단 (필수, **외부 링크 기반**)

> ❗ 1:1 채팅 기능은 **v2 확장 기능으로 제외**

---

### 5-2. 스터디 모집

### 제공 기능

- 모집글 CRUD (작성 / 조회 / 수정 / 삭제)
- 모집 상태 변경 (모집중 / 마감)

### 모집글 입력 항목

- 제목
- 내용
- 모집 인원
- 기간
- 모집 상태
- 연락 수단 (필수)

---

## 6. 자유게시판 기능

자유게시판은 **모집 기능과 완전히 분리된 일반 게시판**이다.

### 6-1. 게시글

- 게시글 CRUD 제공
- 입력 항목:
    - 제목
    - 본문
- 좋아요 기능 제공
- 학과 필터링 가능

> ❗ 모집 상태, 모집 인원 개념 **없음**

---

### 6-2. 댓글

- 댓글 CRUD 제공
- 입력 항목:
    - 본문
- 댓글 좋아요 기능 제공

---

### 6-3. 검색

- 제목 및 본문 키워드 검색
- 학과 필터링

---

## 7. 권한 정책 요약

- 로그인 사용자만:
    - 모집글 작성 / 지원
    - 게시글 작성 / 댓글 작성 / 좋아요
- 작성자 전용 기능:
    - 모집글 수정 / 삭제
    - 모집 상태 변경
    - 지원자 목록 조회
    - 지원 승인 / 거절

---

## 8. 확장 기능 (v2)

다음 기능은 **이번 프로젝트 범위에서 제외**한다.

- 1:1 실시간 채팅
- 알림 시스템

---

## 9. 확정 사항 요약

- 홈 화면은 기능 버튼 중심 구조
- 로그인은 OAuth + 학교 이메일 도메인 제한
- 모집 기능과 자유게시판 도메인 완전 분리
- 모집글에만 모집 상태 / 지원 개념 존재
- 자유게시판은 단순 게시판 구조 유지
