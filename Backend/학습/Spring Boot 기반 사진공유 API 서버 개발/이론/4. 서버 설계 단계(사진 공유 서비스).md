#### 사용자 + 콘텐츠(사진/게시물) + 댓글/좋아요 + 파일 업로드 + 인증

#### 핵심 기능 요구사항
	1. 인증 / 회원
		* 회원가입
		* 로그인(Access Token 발급)
		* 내 정보 조회
		* 로그아웃(선택 : Refresh Token 저장 방식 또는 블랙리스트 정책에 따라)
		* (선택) 토큰 재발급(Refresh Token 기반)

	2. 사진/게시물
		* 게시물 등록(텍스트 + 사진 URL)
		* 게시물 목록 조회(페이지네이션, 최신순 정렬)
		* 게시물 상세 조회
		* 게시물 수정/삭제(작성자 또는 관리자 권한)
		* (선택) 게시물 조회 시 좋아요 수 포함

	3. 댓글
		* 댓글 작성
		* 댓글 목록 조회(게시물 기준, 작성 순 정렬)
		* 댓글 수정/삭제(작성자 또는 관리자 권한)

	4. 좋아요
		* 게시물 좋아요 등록
		* 게시물 좋아요 취소
		* 게시물 좋아요 수 조회(게시물 상세 응답에 포함)
		* (선택) 내가 좋아요한 게시물 여부 확인

	5. 파일 업로드(선택)
		* Presigned URL 발급(API 서버)
		* 클라이언트가 S3로 직접 파일 업로드
		* 업로드 완료 후 게시물 생성 시 해당 URL 저장
		* (선택) 파일 타입/용량 제한 정책

	6. 운영성 기능
		* 스케줄러 작업
			- 만료 토큰 정리
			- 임시 업로드 파일 정리
			- 게시물/좋아요/댓글 통계 집계
		* 서버 시간대 고정(Asia/Seoul)

#### 비기능 요구사항
	 1. 응답 포맷 통일(JSON 기반 공통 응답 구조)
	 2. 예외 처리 표준화(에러 코드 + 메시지 일관화)
	 3. 로깅(SLF4J 기반 로그 관리)
	 4. 보안
	 	- 비밀번호 해시 처리
	 	- JWT 기반 인증/인가
	 5. DB 트랜잭션 관리(서비스 계층 기준)
	 6. 배포
	 	- 단일 Jar 산출
	 	- start/stop 스크립트
	 	- appspec.yml 기반 배포 자동화


#### 데이터 모델(ERD) 설계
테이블/엔티티
	1. User
		* id (PK)
		* email (UK)
		* passwordHash
		* nickname
		* role (USER / ADMIN)
		* createdAt, updatedAt (JPA Auditing)

	2. Post
		* id (PK)
		* authorId (FK → User.id)
		* title
		* content
		* imageUrl (또는 PostImage 테이블로 분리 가능)
		* likeCount (캐시용 컬럼, 선택)
		* createdAt, updatedAt

	3. Comment
		* id (PK)
		* postId (FK → Post.id)
		* authorId (FK → User.id)
		* content
		* createdAt, updatedAt

	4. PostLike
		* id (PK)
		* postId (FK → Post.id)
		* userId (FK → User.id)
		* (postId, userId) UNIQUE 제약
			- 동일 게시물에 대한 중복 좋아요 방지
		* createdAt

	5. RefreshToken / TokenStore (선택)
		* id (PK)
		* userId (FK → User.id)
		* token
		* expiresAt
		* createdAt

인덱스(성능상 보편)
	* User.email UNIQUE
	* Post(authorId, createdAt) 복합 인덱스
		- 작성자별 게시물 목록 조회 최적화
	* Comment(postId, createdAt) 복합 인덱스
		- 게시물별 댓글 목록 조회 최적화
	* PostLike(postId, userId) UNIQUE 인덱스
		- 좋아요 중복 방지 및 여부 조회 최적화


#### API 명세(REST)
1. Auth
	* POST /api/auth/signup
		- 회원가입
	* POST /api/auth/login
		- 로그인 및 Access Token 발급
	* GET /api/users/me (인증 필요)
		- 내 정보 조회
	* (선택) POST /api/auth/refresh
		- Access Token 재발급

2. Posts
	* POST /api/posts (인증)
		- 게시물 등록
	* GET /api/posts?page=0&size=20
		- 게시물 목록 조회(페이지네이션)
	* GET /api/posts/{postId}
		- 게시물 상세 조회
	* PATCH /api/posts/{postId} (작성자/관리자)
		- 게시물 수정
	* DELETE /api/posts/{postId} (작성자/관리자)
		- 게시물 삭제

3. Comments
	* POST /api/posts/{postId}/comments (인증)
		- 댓글 작성
	* GET /api/posts/{postId}/comments
		- 댓글 목록 조회
	* PATCH /api/comments/{commentId} (작성자/관리자)
		- 댓글 수정
	* DELETE /api/comments/{commentId} (작성자/관리자)
		- 댓글 삭제

4. Likes
	* POST /api/posts/{postId}/likes (인증)
		- 게시물 좋아요
	* DELETE /api/posts/{postId}/likes (인증)
		- 게시물 좋아요 취소

5. Upload
	* POST /api/uploads/presign (인증)
		- S3 Presigned URL 발급
