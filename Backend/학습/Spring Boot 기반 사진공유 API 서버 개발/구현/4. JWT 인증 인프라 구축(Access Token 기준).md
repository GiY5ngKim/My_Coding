## 4.0 목표(유지 + 명확화)

- “로그인 성공 → JWT 발급”
    
- “요청 시 JWT 검증 → 인증 객체(SecurityContext) 세팅”
    
- 인증 실패 시에도 **ApiResponse 포맷으로** 401 내려주기
    
- 지금 단계에서는 **도메인 권한 제어는 최소화**(auth만 보호, 나머지는 ‘인증 여부’만 체크)

## 4.2 JWT 설정 값 추가 — 보완(프로퍼티 바인딩 전제)

당신의 yml은 핵심 값이 있습니다. 다만 실사용에서는 **헤더명/프리픽스**도 같이 둬야 필터 구현이 깔끔해집니다.

### 파일 경로: `src/main/resources/application.yml`

```yaml
spring: # Spring 설정 루트
  profiles: # 실행 프로필 설정
    active: local # 기본 프로필 local

server: # 서버 설정 루트
  port: 8080 # 서버 포트 8080

jwt: # JWT 설정 루트
  secret: "CHANGE_ME_LOCAL_SECRET_KEY_32+_CHARS" # 서명 키(로컬용, 32자 이상 권장)
  access-token-expiration-ms: 3600000 # 액세스 토큰 만료(1시간)
  header: "Authorization" # 토큰을 담을 헤더 이름
  token-prefix: "Bearer " # Authorization 헤더 값의 접두사
```

> 운영에서는 secret을 환경변수로 관리하는 게 맞고, 지금은 학습 단계라 yml에 박아도 됩니다.

## 4.3 JwtProperties (yml 바인딩) 추가

> “왜 필요하냐”  
> 문자열로 yml을 매번 직접 읽지 않고, 타입 안정적으로 주입받아 **실수(키 오타/타입 오류)**를 줄입니다.

### 파일 경로: `src/main/java/com/cat/picshare/auth/jwt/JwtProperties.java`

```java
package com.cat.picshare.auth; // auth 패키지 선언이다.

import org.springframework.boot.context.properties.ConfigurationProperties; // yml 바인딩을 위한 임포트이다.

@ConfigurationProperties(prefix = "jwt") // jwt.* 설정을 이 클래스에 바인딩한다.
public class JwtProperties { // JWT 설정을 담는 클래스이다.

    private String secret; // 서명 비밀키이다.
    private long accessTokenExpirationMs; // 토큰 만료시간(ms)이다.
    private String header; // 토큰을 담는 헤더 이름이다.
    private String tokenPrefix; // Bearer 접두사이다.

    public String getSecret() { // secret 값을 반환한다.
        return secret; // secret을 반환한다.
    }

    public void setSecret(String secret) { // secret 값을 주입받는다.
        this.secret = secret; // 필드에 저장한다.
    }

    public long getAccessTokenExpirationMs() { // 만료시간을 반환한다.
        return accessTokenExpirationMs; // 만료시간을 반환한다.
    }

    public void setAccessTokenExpirationMs(long accessTokenExpirationMs) { // 만료시간을 주입받는다.
        this.accessTokenExpirationMs = accessTokenExpirationMs; // 필드에 저장한다.
    }

    public String getHeader() { // 헤더명을 반환한다.
        return header; // 헤더명을 반환한다.
    }

    public void setHeader(String header) { // 헤더명을 주입받는다.
        this.header = header; // 필드에 저장한다.
    }

    public String getTokenPrefix() { // 접두사를 반환한다.
        return tokenPrefix; // 접두사를 반환한다.
    }

    public void setTokenPrefix(String tokenPrefix) { // 접두사를 주입받는다.
        this.tokenPrefix = tokenPrefix; // 필드에 저장한다.
    }
}
```

## 4.4 JwtTokenProvider (토큰 생성/검증) 추가

> “왜 필요하냐”  
> 필터/컨트롤러가 JWT 라이브러리(JJWT)를 직접 다루면 코드가 퍼지고 유지보수가 어려워집니다.  
> 토큰 관련 책임을 Provider 하나로 모아 **단일 책임**을 지키는 게 실무에 가깝습니다.

### 파일 경로: `src/main/java/com/cat/picshare/auth/jwt/JwtTokenProvider.java`

```java
package com.cat.picshare.auth; // auth 패키지 선언이다.

import io.jsonwebtoken.Claims; // JWT 클레임 타입을 사용하기 위한 임포트이다.
import io.jsonwebtoken.Jwts; // JWT 빌더/파서를 사용하기 위한 임포트이다.
import io.jsonwebtoken.SignatureAlgorithm; // 서명 알고리즘을 사용하기 위한 임포트이다.
import io.jsonwebtoken.security.Keys; // HMAC 키 생성을 위한 임포트이다.
import org.springframework.stereotype.Component; // 스프링 컴포넌트 등록을 위한 임포트이다.

import java.nio.charset.StandardCharsets; // 문자열을 바이트로 변환하기 위한 임포트이다.
import java.security.Key; // 서명 키 타입을 사용하기 위한 임포트이다.
import java.util.Date; // 만료시간 계산을 위한 임포트이다.

@Component // 스프링 빈으로 등록한다.
public class JwtTokenProvider { // JWT 생성/검증을 담당하는 클래스이다.

    private final JwtProperties jwtProperties; // JWT 설정을 주입받는다.
    private final Key signingKey; // 서명에 사용할 Key 객체이다.

    public JwtTokenProvider(JwtProperties jwtProperties) { // 생성자이다.
        this.jwtProperties = jwtProperties; // 설정을 저장한다.
        this.signingKey = Keys.hmacShaKeyFor(jwtProperties.getSecret().getBytes(StandardCharsets.UTF_8)); // HMAC 키를 생성한다.
    }

    public String createAccessToken(String subject) { // Access Token을 생성한다.
        Date now = new Date(); // 현재 시간을 만든다.
        Date expiry = new Date(now.getTime() + jwtProperties.getAccessTokenExpirationMs()); // 만료 시간을 계산한다.
        return Jwts.builder() // 토큰 빌더를 시작한다.
                .setSubject(subject) // subject(예: email 또는 userId)를 넣는다.
                .setIssuedAt(now) // 발급 시간을 넣는다.
                .setExpiration(expiry) // 만료 시간을 넣는다.
                .signWith(signingKey, SignatureAlgorithm.HS256) // HS256으로 서명한다.
                .compact(); // 문자열 토큰으로 만든다.
    }

    public boolean validateToken(String token) { // 토큰 유효성을 검증한다.
        try { // 예외 가능 구간을 감싼다.
            Jwts.parserBuilder() // 파서 빌더를 만든다.
                    .setSigningKey(signingKey) // 서명 키를 설정한다.
                    .build() // 파서를 만든다.
                    .parseClaimsJws(token); // 파싱이 되면 유효한 토큰이다.
            return true; // 검증 성공을 반환한다.
        } catch (Exception e) { // 서명 불일치/만료/형식 오류 등 모든 예외를 처리한다.
            return false; // 검증 실패를 반환한다.
        }
    }

    public String getSubject(String token) { // 토큰에서 subject를 뽑는다.
        Claims claims = Jwts.parserBuilder() // 파서 빌더를 만든다.
                .setSigningKey(signingKey) // 서명 키를 설정한다.
                .build() // 파서를 만든다.
                .parseClaimsJws(token) // 토큰을 파싱한다.
                .getBody(); // 클레임 바디를 얻는다.
        return claims.getSubject(); // subject를 반환한다.
    }
}
```

## 4.5 JwtAuthenticationFilter (요청 시 JWT 검증 → SecurityContext 세팅)

> “왜 필요하냐”  
> Spring Security는 `SecurityContext`에 인증 객체가 있어야 “인증된 사용자”로 인식합니다.  
> 따라서 요청마다 JWT를 검증해서 `SecurityContext`를 채워주는 필터가 필수입니다.

> 주의: 지금 단계는 “User 도메인 없음” 전제이므로  
> subject를 username처럼 취급하고 권한은 최소값으로 넣습니다(ROLE_USER 고정).

### 파일 경로: `src/main/java/com/cat/picshare/auth/jwt/JwtAuthenticationFilter.java`

```java
package com.cat.picshare.auth; // auth 패키지 선언이다.

import jakarta.servlet.FilterChain; // 필터 체인 타입을 사용하기 위한 임포트이다.
import jakarta.servlet.ServletException; // 서블릿 예외 타입을 사용하기 위한 임포트이다.
import jakarta.servlet.http.HttpServletRequest; // HTTP 요청 타입을 사용하기 위한 임포트이다.
import jakarta.servlet.http.HttpServletResponse; // HTTP 응답 타입을 사용하기 위한 임포트이다.
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken; // 인증 객체 타입을 사용하기 위한 임포트이다.
import org.springframework.security.core.authority.SimpleGrantedAuthority; // 단순 권한 객체를 사용하기 위한 임포트이다.
import org.springframework.security.core.context.SecurityContextHolder; // SecurityContext를 제어하기 위한 임포트이다.
import org.springframework.web.filter.OncePerRequestFilter; // 요청당 1회 실행되는 필터를 사용하기 위한 임포트이다.

import java.io.IOException; // IO 예외를 처리하기 위한 임포트이다.
import java.util.List; // 권한 리스트를 만들기 위한 임포트이다.

public class JwtAuthenticationFilter extends OncePerRequestFilter { // 요청당 1회 실행되는 JWT 인증 필터이다.

    private final JwtTokenProvider tokenProvider; // 토큰 생성/검증기를 가진다.
    private final JwtProperties jwtProperties; // 헤더명/프리픽스 설정을 가진다.

    public JwtAuthenticationFilter(JwtTokenProvider tokenProvider, JwtProperties jwtProperties) { // 생성자이다.
        this.tokenProvider = tokenProvider; // Provider를 저장한다.
        this.jwtProperties = jwtProperties; // Properties를 저장한다.
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain) throws ServletException, IOException { // 필터 본체이다.
        String headerValue = request.getHeader(jwtProperties.getHeader()); // Authorization 헤더 값을 읽는다.
        if (headerValue != null && headerValue.startsWith(jwtProperties.getTokenPrefix())) { // Bearer 접두사가 있으면 토큰으로 본다.
            String token = headerValue.substring(jwtProperties.getTokenPrefix().length()); // 접두사를 제거하고 토큰만 뽑는다.
            if (tokenProvider.validateToken(token)) { // 토큰이 유효하면 인증 객체를 만든다.
                String subject = tokenProvider.getSubject(token); // subject를 꺼낸다.
                List<SimpleGrantedAuthority> authorities = List.of(new SimpleGrantedAuthority("ROLE_USER")); // 지금 단계는 권한을 최소로 고정한다.
                UsernamePasswordAuthenticationToken authentication = new UsernamePasswordAuthenticationToken(subject, null, authorities); // 인증 객체를 만든다.
                SecurityContextHolder.getContext().setAuthentication(authentication); // SecurityContext에 인증 객체를 세팅한다.
            }
        }
        filterChain.doFilter(request, response); // 다음 필터 또는 컨트롤러로 진행한다.
    }
}
```

## 4.6 인증 실패 시 ApiResponse로 401 내려주기 (EntryPoint 구현)

당신이 `AuthenticationEntryPointImpl`을 걸어둔 것은 방향이 정확합니다.  
다만 “공통 응답(ApiResponse/ApiError)”을 실제로 쓰려면 **EntryPoint에서 JSON을 직접 써야** 합니다.

아래는 “C002 / 인증이 필요합니다”를 공통 포맷으로 내려주는 구현입니다.

### 파일 경로: `src/main/java/com/cat/picshare/auth/handler/AuthenticationEntryPointImpl.java`

```java
package com.cat.picshare.auth; // auth 패키지 선언이다.

import com.cat.picshare.common.response.ApiResponse; // 공통 응답을 사용하기 위한 임포트이다.
import com.cat.picshare.common.response.ApiError; // 공통 에러를 사용하기 위한 임포트이다.
import com.fasterxml.jackson.databind.ObjectMapper; // JSON 직렬화를 위한 임포트이다.
import jakarta.servlet.ServletException; // 서블릿 예외 타입을 사용하기 위한 임포트이다.
import jakarta.servlet.http.HttpServletRequest; // 요청 타입을 사용하기 위한 임포트이다.
import jakarta.servlet.http.HttpServletResponse; // 응답 타입을 사용하기 위한 임포트이다.
import org.springframework.http.MediaType; // Content-Type 설정을 위한 임포트이다.
import org.springframework.security.core.AuthenticationException; // 인증 예외 타입을 사용하기 위한 임포트이다.
import org.springframework.security.web.AuthenticationEntryPoint; // EntryPoint 인터페이스를 사용하기 위한 임포트이다.

import java.io.IOException; // IO 예외를 처리하기 위한 임포트이다.

public class AuthenticationEntryPointImpl implements AuthenticationEntryPoint { // 인증 실패 시 401을 내려주는 EntryPoint이다.

    private final ObjectMapper objectMapper = new ObjectMapper(); // JSON 변환기를 만든다.

    @Override
    public void commence(HttpServletRequest request, HttpServletResponse response, AuthenticationException authException) throws IOException, ServletException { // 인증 실패 시 호출된다.
        ApiError error = new ApiError("C002", "인증이 필요합니다"); // 에러 객체를 만든다(당신의 예시 포맷을 따른다).
        ApiResponse<Void> body = ApiResponse.fail(error); // 공통 응답 포맷으로 감싼다.
        response.setStatus(HttpServletResponse.SC_UNAUTHORIZED); // HTTP 401로 설정한다.
        response.setContentType(MediaType.APPLICATION_JSON_VALUE); // JSON 응답임을 설정한다.
        response.setCharacterEncoding("UTF-8"); // 한글 깨짐을 방지하기 위해 UTF-8을 설정한다.
        response.getWriter().write(objectMapper.writeValueAsString(body)); // 바디를 JSON 문자열로 써서 내려준다.
    }
}
```

> 위 코드가 컴파일되려면 `ApiResponse.fail(...)` / `ApiError` 생성자가 실제 구현과 맞아야 합니다.  
> 당신 Chapter 2 구현체가 다르면, 여기서 **fail 생성 방식만** 당신 코드에 맞게 바꿔주면 됩니다(나머지 흐름은 동일).

## 4.7 SecurityConfig (수정본: 패키지 구조/빈 등록/필터 체인 정리)

당신 코드에서 보완할 점은 다음입니다.

- `JwtProperties`를 활성화하지 않으면 yml 바인딩이 안 됩니다.
    
- `JwtAuthenticationFilter`를 `@Bean`으로 만들 때 `JwtProperties`도 같이 주입해야 합니다(위 필터 구현 기준).
    
- `SecurityConfig` 패키지는 확정 구조상 `config` 아래가 자연스럽습니다(세부 폴더는 프로젝트 취향이지만 “확정된 구조” 기준으로 맞춥니다).
    

### 파일 경로: `src/main/java/com/cat/picshare/config/SecurityConfig.java`

```java
package com.cat.picshare.config; // config 패키지 선언이다.

import com.cat.picshare.auth.AuthenticationEntryPointImpl; // 401 JSON 처리를 위한 EntryPoint를 사용하기 위한 임포트이다.
import com.cat.picshare.auth.JwtAuthenticationFilter; // JWT 필터를 사용하기 위한 임포트이다.
import com.cat.picshare.auth.JwtProperties; // JWT 설정 바인딩 클래스를 사용하기 위한 임포트이다.
import com.cat.picshare.auth.JwtTokenProvider; // JWT Provider를 사용하기 위한 임포트이다.
import org.springframework.boot.context.properties.EnableConfigurationProperties; // ConfigurationProperties 활성화를 위한 임포트이다.
import org.springframework.context.annotation.Bean; // Bean 등록을 위한 임포트이다.
import org.springframework.context.annotation.Configuration; // 설정 클래스 표시를 위한 임포트이다.
import org.springframework.security.config.annotation.web.builders.HttpSecurity; // HttpSecurity 설정을 위한 임포트이다.
import org.springframework.security.config.http.SessionCreationPolicy; // 무상태 세션 정책을 위한 임포트이다.
import org.springframework.security.web.SecurityFilterChain; // 필터 체인 타입을 위한 임포트이다.
import org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter; // 필터 순서 지정을 위한 임포트이다.

@Configuration // 설정 클래스임을 선언한다.
@EnableConfigurationProperties(JwtProperties.class) // JwtProperties 바인딩을 활성화한다.
public class SecurityConfig { // Spring Security 설정 클래스이다.

    @Bean // JWT 필터를 스프링 빈으로 등록한다.
    public JwtAuthenticationFilter jwtAuthenticationFilter(JwtTokenProvider tokenProvider, JwtProperties jwtProperties) { // 필터 빈 생성 메서드이다.
        return new JwtAuthenticationFilter(tokenProvider, jwtProperties); // 필터를 생성해 반환한다.
    }

    @Bean // SecurityFilterChain을 스프링 빈으로 등록한다.
    public SecurityFilterChain securityFilterChain(HttpSecurity http, JwtAuthenticationFilter jwtFilter) throws Exception { // 보안 구성을 정의한다.
        http.csrf(csrf -> csrf.disable()); // REST API는 일반적으로 CSRF를 끈다(세션을 안 쓰기 때문).
        http.sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS)); // 세션을 사용하지 않는 무상태로 설정한다.
        http.formLogin(form -> form.disable()); // 폼 로그인 페이지를 비활성화한다.
        http.httpBasic(basic -> basic.disable()); // Basic 인증을 비활성화한다.
        http.exceptionHandling(ex -> ex.authenticationEntryPoint(new AuthenticationEntryPointImpl())); // 인증 실패 시 401 JSON을 내려주게 설정한다.
        http.authorizeHttpRequests(auth -> auth // 인가 규칙을 설정한다.
                .requestMatchers("/api/auth/**").permitAll() // 로그인/회원가입은 누구나 접근 가능하다.
                .requestMatchers("/api/test/**").permitAll() // 테스트는 지금 단계에서 열어둔다(추후 제거).
                .anyRequest().authenticated() // 나머지 요청은 인증을 요구한다.
        );
        http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class); // UsernamePasswordAuthenticationFilter 앞에 JWT 필터를 둔다.
        return http.build(); // 필터 체인을 빌드해 반환한다.
    }
}
```

### 파일 경로: `src/main/java/com/cat/picshare/auth/controller/AuthController.java`

```java
package com.cat.picshare.auth; // auth 패키지 선언이다.

import com.cat.picshare.auth.dto.AuthLoginRequest; // 로그인 요청 DTO를 사용하기 위한 임포트이다.
import com.cat.picshare.auth.dto.AuthLoginResponse; // 로그인 응답 DTO를 사용하기 위한 임포트이다.
import com.cat.picshare.common.response.ApiResponse; // 공통 응답을 사용하기 위한 임포트이다.
import jakarta.validation.Valid; // 유효성 검증 트리거를 위한 임포트이다.
import org.springframework.web.bind.annotation.PostMapping; // POST 매핑을 위한 임포트이다.
import org.springframework.web.bind.annotation.RequestBody; // 요청 바디 바인딩을 위한 임포트이다.
import org.springframework.web.bind.annotation.RequestMapping; // 공통 경로 지정을 위한 임포트이다.
import org.springframework.web.bind.annotation.RestController; // REST 컨트롤러 선언을 위한 임포트이다.

@RestController // REST 컨트롤러임을 선언한다.
@RequestMapping("/api/auth") // auth API의 공통 경로를 지정한다.
public class AuthController { // 인증 관련 컨트롤러이다.

    private final JwtTokenProvider tokenProvider; // 토큰 Provider를 주입받는다.

    public AuthController(JwtTokenProvider tokenProvider) { // 생성자이다.
        this.tokenProvider = tokenProvider; // Provider를 저장한다.
    }

    @PostMapping("/login") // 로그인 엔드포인트이다.
    public ApiResponse<AuthLoginResponse> login(@Valid @RequestBody AuthLoginRequest request) { // 로그인 요청을 처리한다.
        String subject = request.getEmail(); // 지금 단계에서는 email을 subject로 사용한다.
        String token = tokenProvider.createAccessToken(subject); // Access Token을 발급한다.
        return ApiResponse.ok(new AuthLoginResponse(token)); // 공통 응답 포맷으로 토큰을 내려준다.
    }
}
```

## 4.9 테스트 엔드포인트 정리(“실제로 존재하는 URL”로 검증)

당신이 작성한 `/api/test/custom`, `/api/test/auth` 흐름은 좋습니다.  
다만 “보호 URL”은 **컨트롤러가 실제로 존재하는 게 테스트가 더 명확합니다.**

### 파일 경로: `src/main/java/com/cat/picshare/common/TestController.java`

(패키지는 `common`으로 두겠습니다. “테스트용 컨트롤러”이기 때문입니다.)

```java
package com.cat.picshare.common; // common 패키지 선언이다.

import com.cat.picshare.common.response.ApiResponse; // 공통 응답을 사용하기 위한 임포트이다.
import org.springframework.security.core.Authentication; // 현재 인증 객체를 받기 위한 임포트이다.
import org.springframework.web.bind.annotation.GetMapping; // GET 매핑을 위한 임포트이다.
import org.springframework.web.bind.annotation.RequestMapping; // 경로 지정을 위한 임포트이다.
import org.springframework.web.bind.annotation.RestController; // REST 컨트롤러 선언을 위한 임포트이다.

@RestController // REST 컨트롤러임을 선언한다.
@RequestMapping("/api/test") // 테스트 API 공통 경로이다.
public class TestController { // 테스트용 컨트롤러이다.

    @GetMapping("/custom") // 강제 예외 테스트 엔드포인트이다.
    public ApiResponse<Void> custom() { // 예외 핸들링 동작을 확인한다.
        throw new RuntimeException("강제 예외 발생"); // 일부러 예외를 터뜨린다(당신의 CustomException 흐름이면 그 예외로 교체).
    }

    @GetMapping("/auth") // 인증이 필요한 테스트 엔드포인트이다.
    public ApiResponse<String> auth(Authentication authentication) { // 인증 객체를 주입받아 확인한다.
        String principal = authentication != null ? String.valueOf(authentication.getPrincipal()) : "null"; // principal을 문자열로 만든다.
        return ApiResponse.ok("AUTH OK, principal=" + principal); // 인증 성공 시 principal을 포함해 반환한다.
    }
}
```

> `/api/test/auth`는 SecurityConfig에서 `.anyRequest().authenticated()`에 걸리므로  
> 토큰 없으면 401(JSON), 토큰 있으면 200(JSON)로 동작해야 정상입니다.

# 4.10 Chapter 4 동작 확인(정확 버전)

## (1) 인증 없이 접근 가능해야 하는 URL (permitAll)

URL: `http://localhost:8080/api/test/custom`

기대 결과(당신 의도 유지):

- HTTP 400(혹은 당신 ErrorCode 설계에 따라 500일 수도 있음)
    
- JSON(ApiResponse) 형태
    
- GlobalExceptionHandler가 처리
    

## (2) 인증이 필요한 URL (JWT 보호 구간)

URL: `http://localhost:8080/api/test/auth`

### 토큰 없이 요청

- 기대 결과: **HTTP 401**
    
- HTML/로그인 화면이 아니라 **JSON(ApiResponse)**
    

### 토큰 발급 후 요청

1. 로그인으로 토큰 받기
    

- `POST http://localhost:8080/api/auth/login`
    
- Body: `{"email":"a@a.com","password":"1111"}`
    

2. 토큰으로 보호 API 호출
    

- `GET http://localhost:8080/api/test/auth`
    
- Header: `Authorization: Bearer <accessToken>`
    
- 기대 결과: **HTTP 200**
    
- JSON(ApiResponse)로 `"AUTH OK, principal=a@a.com"` 같은 메시지

# 4.11 “컨트롤러가 없어도 401이 의미 있나?” (정확한 설명)

당신 설명은 **완전히 틀리진 않지만 조건이 붙습니다.**

- Spring Security 필터 체인은 DispatcherServlet(컨트롤러 매핑)보다 앞에서 실행되므로,  
    **해당 URL이 인증이 필요하다고 판단되면** 컨트롤러가 없어도 먼저 401을 낼 수 있습니다.
    
- 하지만, 실무/학습에서 혼동이 많아서 이 단계에서는  
    **“존재하는 엔드포인트로 401/200을 확실히 검증”**하는 편이 더 안전합니다.
    
- 즉, “컨트롤러 없어도 된다”는 논리로 가면  
    permitAll/매핑/리소스 핸들러 등 변수 때문에 404가 섞여서 디버깅이 어려워집니다.

## 사실확인 1차(당신 문서의 결론 검증)

`/api/test/custom`에서 JSON(ApiResponse) 에러가 떨어지는 것은 “컨트롤러 진입 + 예외 발생 + GlobalExceptionHandler가 ApiResponse로 변환”의 파이프라인이 정상이라는 의미가 맞습니다.  
다만 HTTP 상태코드는 당신의 ErrorCode/예외 매핑 정책에 따라 400이 아닐 수도 있으니(예: RuntimeException을 500으로 보내는 설계), **상태코드는 ‘정책 일치 여부’로 판단**하는 게 정확합니다.

## 사실확인 2차(“JWT 먼저 구축 → 도메인 구현” 순서의 타당성)

JWT 필터에서 `SecurityContext`가 세팅되는 구조를 먼저 확정해두면, 이후 Post/Comment에서 “작성자 검증(본인만 수정/삭제)” 같은 로직이 `Authentication` 기반으로 자연스럽게 붙어서 **재작업이 줄어드는 흐름**이 맞습니다. 즉, 이 순서는 실무에서도 흔합니다.
