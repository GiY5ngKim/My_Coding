## 1) 면접 전에 최소로 알고 있어야 할 핵심 맵

### A. Spring Core (거의 무조건)

- **IoC/DI**: 왜 쓰는지(결합도↓, 테스트 용이, 확장성↑), DI 방식(생성자/세터/필드)
    
- **Bean / ApplicationContext**: 빈 등록(컴포넌트 스캔 vs @Bean), 라이프사이클(init/destroy), 스코프(singleton/prototype/request/session)
    
- **AOP**: 어디에 쓰는지(로깅/트랜잭션/보안), 프록시 기반(대충이라도 “프록시로 가로챈다” 설명 가능해야 함)
    

### B. Web (Spring MVC / REST)

- **요청 흐름**: Controller → Service → Repository, 그리고 **DispatcherServlet**이 앞에서 라우팅
    
- **@RestController vs @Controller** / **@ResponseBody**
    
- **DTO 사용 이유**(엔티티 직접 노출 X), **Validation(@Valid, BindingResult)**, 예외 처리(**@ControllerAdvice**)
    
- **REST 설계 감각**: URI, HTTP Method, 상태코드(200/201/204/400/401/403/404/409/500)
    

### C. DB/JPA (동아리도 여기서 컷 많이 남)

- **JPA 영속성 컨텍스트**: 1차 캐시, 변경감지(dirty checking), 지연로딩(LAZY) 이유
    
- **N+1**이 뭔지, 해결 방향(fetch join, EntityGraph, batch size 등 “키워드”라도)
    
- **트랜잭션(@Transactional)**: 경계 설정, 롤백 조건(RuntimeException), 읽기전용 옵션
    
- **연관관계**: 주인(owner), mappedBy 의미, 양방향 설계 시 주의
    

### D. 실전 기본기 (경험 질문으로 연결됨)

- **로그로 디버깅**(에러 원인 추적 방법)
    
- **테스트**: 단위/통합 차이, SpringBootTest, MockMvc, Mockito(“써봤다/왜 쓴다” 수준)
    
- **빌드/환경**: Gradle/Maven, application.yml 프로파일(dev/prod)

## 2) “거의 무조건 나오는” 질문 리스트 (의도까지)

아래는 질문 + 면접관이 듣고 싶은 포인트를 같이 적어둘게.

### Spring Core

1. **스프링을 왜 쓰나요?**
    
    - 프레임워크가 제공하는 핵심 가치: DI로 구조화, 생산성, 생태계, 테스트/확장
        
2. **IoC/DI가 뭔가요? 필드 주입 vs 생성자 주입?**
    
    - 결론: **생성자 주입 선호**(불변/테스트 용이/순환참조 조기 발견)
        
3. **Bean이 뭐고 언제 생성돼요?**
    
    - 기본: 싱글톤, 컨테이너 시작 시 생성(대부분), 라이프사이클 언급 가능하면 플러스
        
4. **AOP가 뭔가요? 어디에 써봤나요?**
    
    - 로깅/트랜잭션/권한 체크 “횡단 관심사” 분리, 프록시 개념
        

### Spring MVC / REST

5. **요청이 들어오면 스프링에서 어떤 흐름으로 처리되나요?**
    
    - DispatcherServlet → HandlerMapping → Controller → View/Response
        
6. **@RestController와 @Controller 차이?**
    
    - 반환이 View냐 Body(JSON)냐
        
7. **DTO를 왜 쓰나요?**
    
    - 엔티티 노출 위험, 스펙 변화 대응, 검증/직렬화 문제 분리
        
8. **상태코드 201/204는 언제?**
    
    - 생성: 201(+Location), 삭제/수정 후 body 없음: 204
        
9. **예외 처리는 어떻게 설계하나요?**
    
    - @ControllerAdvice, 공통 에러 응답 포맷, 유효성 에러 처리
        

### JPA / 트랜잭션

10. **영속성 컨텍스트가 뭐예요?**
    
    - 1차 캐시/변경감지/쓰기 지연/동일성 보장
        
11. **N+1이 뭔가요? 왜 생기고 어떻게 막아요?**
    
    - LAZY + 반복 접근, fetch join/EntityGraph 등
        
12. **@Transactional이 붙으면 뭐가 달라져요?**
    
    - 커밋/롤백 경계, Lazy loading 가능 타이밍, 읽기전용 최적화
        
13. **양방향 연관관계에서 mappedBy가 의미하는 건?**
    
    - “누가 FK를 관리하는가(주인)”, 주인이 아닌 쪽은 읽기 전용 성격
        

### 실전/프로젝트

14. **가장 크게 막혔던 버그와 해결 과정은?**
    
    - “로그→재현→원인 가설→검증→해결→회고” 프로세스
        
15. **성능 문제를 겪어봤나요?**
    
    - 쿼리 최적화/N+1/인덱스/캐시/페이징 중 하나라도 경험 연결
        
16. **협업은 어떻게 했나요?**
    
    - Git flow, PR, 코드리뷰, 이슈 관리(간단히라도)

## 3) 동아리 면접에서 특히 자주 나오는 “현장형” 질문

- **“왜 스프링 파트 지원했어요?”**
    
    - 단순 흥미 말고: 웹 요청-응답, 구조 설계, DB 연동, 협업 경험 쌓고 싶다
        
- **“우리 동아리에서 뭘 해보고 싶어요?”**
    
    - 예: 로그인/게시판/모집 기능/검색/필터/권한 같은 구체 기능 제안
        
- **“시간 투자 얼마나 가능?”**
    
    - 주 몇 시간, 시험 기간 조정 방식까지 현실적으로
        
- **“Spring Boot로 뭐 만들어봤어?”**
    
    - 작은 프로젝트라도: 기능/아키텍처/문제해결 1개씩 말할 준비
        
- **“JPA 쓸 때 주의점 뭐야?”**
    
    - N+1, 연관관계, 트랜잭션 범위, 엔티티 직렬화(순환참조) 같은 키워드
