##### 서버(server) : 네트워크를 통해 받은 요청을 처리 후, 응답을 반환하는 프로그램
##### 클라이언트(client) : 요청을 보내는쪽(브라우저, 앱, Postman 등)

서버는 다음 단계에 따라 작업을 수행한다.
1. 요청 : 누가(클라이언트) 어떤 주소(URL)로 어떤 방식(GET/POST 등)으로 요청했는지 받음
2. 처리 : 로직 실행(로그인 확인, 데이터 저장, 조회, 계산 등
3. 응답 : 결과를 HTML, JSON, 파일, 상태코드 등의 형태로 반환

서버 포트 : Spring(8080), MySQL(3306), Redis(6379)

#### URL이란? (http://localhost:8080/health)
* http://      통신 규칙(프로토콜)
* localhost   호스트(어떤 컴퓨터?)
* :8080       포트(어떤 프로그램?)
* /health     경로(그 프로그램의 어떤 기능?)

#### HTTP란 무엇인가?
* 웹에서 가장 간단한 통신규칙 : 요청 1번 보내면 응답 1번 받는다.

##### HTTP의 요청
- 메서드(method) : 무엇을 하고 싶은지
	- GET : 가져와라
	- POST : 만들어라 / 보내라
	- PUT/PATCH : 수정해라
	- DELETE : 삭제해라
- 경로(path) : 어떤 대상인지
	- /rooms , /message
- 헤더(header) : 부가 정보(인증, 콘텐츠 타입 등)
- 바디(body) : 실제 데이터(POST에서 주로 사용)

##### HTTP의 응답
* 상태 코드(status code) 결과 상태(200, 404, 500 등)
* 응답 바디 : JSON, HTML 등

##### 백엔드 개발자는 일반적으로 HTTP 서버를 만든다. 서버는 다음을 담당한다
* 데이터 저장(DB)
* 로직 처리(서비스)
* 인증/권한(로그인)
* 외부 시스템 연동(결제, 문자, 이메일)
